Human Wisdom Program

UX Design | UX Research | Development

An online course that provides a road map for everyone to explore and understand themselves more deeply.

Roles and Responsibilities

User Research: User Interviews, Competetive Analysis, Journey Mapping

UX Design: Sketches, Wireframes, High fidelity prototype.

Development: PWA for users and web app for admin, database optimization.

Project Context

Aug 2020 - Sept 2021

Team: Dr Manoj Krishna, Mahesh Ramanathan, Pratik Mangtani, Nidhi Singhvi, Sana Sayyed

Tools Used

Figma
Angular
HTML, CSS
MySQL

Overview

Dr Manoj Krishna is author of, “Understanding Me, Understanding You - an enquiry to be human, and Understanding and Managing Stress”. He aims to scale the benefits of wisdom through a structured online application.

The Human Wisdom Program (HWP) is an e learning platform that helps people discover themselves so they can be happy and live with a sense of peace.

Through this program, we are trying to help people cope up with their problems by empowering them to use self-awareness for solving their problems.

Problem

Dr Manoj Krishna approached us with his book and some course slides on Articulate. The course revolved around,”Five Circles of Wisdom”. There was no structure to the learning material. Dr Krishna wanted a structured and interactive E-learning application that would guide users through the central concept of the five circles of wisdom.

Design Process

I went through the course content on Articulate and read the book to better undertsand the requirements of the stakeholder. I interviewed the founder, Dr Manoj Krishna along with the Chief Technology Officer. Following are some questions that I prepared for the interview:

1. What is your vision behind the learning program?

2. What impact would you like the course to leave on learners?

User Interview

The CTO and I interviewed Dr Manoj Krishna to better understand his vision of the learning application. These intervies gave me an idea of something I could work with for the User Experience

Insights from Interviews

Based on user interviews, I decided to combine features of E-learning and streaming applications. This would wnable users to engage with the learning program, track their progress and the navigation experience would give them an experience of using a digital streaming service.

Competitive Analysis

I decided to carry out competitive analysis with e-learning and streaming applications for inspiration for HWP.

Features

Based on my research, I decided to design a dashboard that would cover the content. We divided the learning material into modules and sections. The dashboard would allow users to easily navigate from one module to another.

Feedback

For progress tracking, we introduced a progress bar at the top. The development of this tracker was challenging because our database did not accommodate sessions to manage page-wise progress. Implementation of the this feature was time-consuming because a collection of pages i.e sessions had to stored in the database.

We introduced a share icon on each screen. Shared screens can be viewed for free by recepients. We decided to have this feature to raise awareness about the program. This feature led us to define access rules for subscribers and non subscribers.
Note: This is covered in the marketing case study of HWP.

We introduced an introduction module for users to get acquainted with the application. This module included different features of the application and guidelines for users to enagage with HWP.

Solution

Development

Database design

Based on our research, list of features and dashboard design, we designed a database. This database was implemented in MySql. We implemented our own Learning Management System for the Human Wisdom Program. This was an iterative process because we had to ensure that the design was scalable and avoided redundant storage of data.

Admin Module

The application has many stakeholders: Super admin, admin, moderator of the discussion forum, content manager, program affiliate. I designed a developed screens for the admin module to manage data on the course. We prepared screens for CRUD operations, reports, pricing, approval of posts. Access to screens based on roles was controlled using Typescript.

Reports

Feedback survey report

We desgined a feedback survey for users after each module. Results of the feedback survey would help us improve the quality of content. I designed the feedback survey report for the admin module based on selections of users.

Wisdom Survey Report

The wisdom survey is available for users on the learning application. I designed a report to analyze results of wisdom surveys to gauge the impact of the Human Wisdom Program on people.

Takeaway

Tradeoffs between technical requirements, design decisions and financial resources.

We had limited financial resources. I was the lead UX Engineer and every design decision had to go through multiple discussions regarding feasibility of implementation. This was an immense learning experience for me. II dealt with multiple iterations of features that involved changes to optimize the database, webservices and software.

Strategic Design

My background in Computer Science played a major role in taking design decisions. I was responsible for designing screens where I considered the feasibility of development. I collaborated with the product manager to estimate time and efforts required to implement designs. These designs were then handed over to our visual designer.

Comfort with ambiguity

There were multiple instances where we were uncertain about design and implementation. It was tempting to design and develop all features with utmost precision. However, due to limited time and resources, we had to deliver at a fast pace. My team and I decided to start low and invest on features according to users’ feedback.
