Background

Understanding Me, Understanding You - Dr Manoj Krishna

Dr Manoj Krishna is author of, “Understanding Me, Understanding You - an enquiry to be human, and Understanding and Managing Stress”. He aims to scale the benefits of wisdom through a structured online application.

Initial Course Content on Articulate

The Human WIsdom Program was created on Articulate slides. The initial requirement of the client was to improve navigation between different sections and development of a portal through which the content can be modified.

Overview

Problem

The learning material on Articulate was unstructured. The content was repetitive, the slides were not interactive and the navigation from one module to the other was difficult. The goal of this project was to design and develop an interactive E-Learning application through which learners could easily navigate between modules and track their progress.

Stakeholders

User Needs

From the interviews we understood the following needs:

1. The client required a design that could allow them to modify the content of the learning application.

2. The solution should be built such that it could be scaled up to for future modifications.

3. The content of the learning application should be structured for easier navigation between modules.

4. A discussion forum for learners to interact with each other and discuss mental health issues.

Supporting quotes

Problem Definition

Based on our interviews, and insights, we decided that our application would address three problems:

To structure the content and make the application more interactive

To connect learners with each other.

To allow ease of navigation between different modules.

Ideation

Based on user interviews, I decided to combine features of E-learning and streaming applications. This would enable users to engage with the learning program, track their progress and the navigation experience would give them an experience of using a digital streaming service. Below are some advantages of E-learning apps and streaming apps.

E-learning app features:

1. Progress Tracking

2. Interaction with peers

3. Interactive course content with assessments.

Streaming app features:

1. Familiar card layout that from popular streaming apps like Netflix and Youtube.

2. Positive transfer of training if users start using HWP.

Feedback

For progress tracking, we introduced a progress bar at the top. The development of this tracker was challenging because our database did not accommodate sessions to manage page-wise progress. Implementation of the this feature was time-consuming because a collection of pages i.e sessions had to stored in the database.

We introduced a share icon on each screen. Shared screens can be viewed for free by recepients. We decided to have this feature to raise awareness about the program. This feature led us to define access rules for subscribers and non subscribers.

Note: This is covered in the marketing case study of HWP.

We introduced an introduction module for users to get acquainted with the application. This module included different features of the application and guidelines for users to enagage with HWP.

Future Modifications

Building the Human Wisdom Community

We decided to build a community of learners and coaches. We achieved this with a discussion forum where users would interact with each other and discuss issues about mental health. We also developed a coaching program through which users could get in touch with psychologists to seek help. Onboarding coaches from all over the world is a future modification.

Module Suggestions based on users’ choices

Pointing users to resources and personalizing their experiences is a future modification. Introducing machine learning to the system for recommendations is a future goal.

Takeaway

Tradeoffs between technical requirements, design decisions and financial resources.

We had limited financial resources. I was the lead UX Engineer and every design decision had to go through multiple discussions regarding feasibility of implementation. This was an immense learning experience for me. II dealt with multiple iterations of features that involved changes to optimize the database, webservices and software.

Strategic Design

My background in Computer Science played a major role in taking design decisions. I was responsible for designing screens where I considered the feasibility of development. I collaborated with the product manager to estimate time and efforts required to implement designs. These designs were then handed over to our visual designer.

Comfort with ambiguity

There were multiple instances where we were uncertain about design and implementation. It was tempting to design and develop all features with utmost precision. However, due to limited time and resources, we had to deliver at a fast pace. My team and I decided to start low and invest on features according to users’ feedback.
