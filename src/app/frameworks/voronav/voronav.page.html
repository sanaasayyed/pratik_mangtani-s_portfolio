Research

Present models of Indoor positioning use Wifi and triangulation technique which is not energy efficient and lacks accuracy. Our initial plan was to have an indoor navigation system. We realized that the main problem lied in efficiently locating the source. We thus based our problem statement on indoor positioning.

Problem

After literature review and surveys, we decided to base our project on finding an energy efficient algorithm to locate users in indoor premises. Currentl systems like triangulation and RFID reequire a lot of equipments. It is difficult to build and maintain.

Algorithm and Solution

Previous Next

After understanding the problem carefully we realized that we needed a dynamic system to position users in an energy efficient way. That is how we came up with Voronav, which is an Indoor Positioning application.

It calculates the current position of a user using Bluetooth Low energy beacons.â€‹

The algorithm made use of a order k voronoi diagram which maps the environment of the user and then creates zones for detecting the position.â€‹

The relative distance of the user from the beacons was calculated and the position of the user was thus detected.

App Flow

Previous Next

We designed an app flow to deploy our system in malls. We created master screens and webservices for easy installation of the system. The application for end users got data from MySql database and real-time mapping through Voronoi algorithm.

Impact and other applications

Previous Next

This scalable solution can be used to address position detection problems in many indoor premises such as museums, hospitals, and malls.

Impact:

This application and the algorithm was further developed to build a nearest nurse detection system for Ruby Hall Clinic, Pune, Maharashtra.

This app is currently used by all branches of Ruby Hall with approximately 1000 users, 650 doctors and 300 consultant doctors.

Tradeoffs between technical constraints and the required user experience

During the research we realized that accurate positioning was not required as long as users comfortably reached their destinations. We, thus used Voronoi mapping and the concept of calculing relative distances.
